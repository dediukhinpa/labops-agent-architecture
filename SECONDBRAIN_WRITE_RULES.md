# second_brain — что и когда записывать (общая политика)

> RED-зона: правит оператор. Единый источник правды, @-импортится в CLAUDE.md всех агентов. Ролевые акценты — в CLAUDE.md каждого агента; здесь общая база.

## Дисциплина (4 правила)
1. **recall ПЕРЕД записью.** Сначала `second_brain-memory_router.recall(...)` — вдруг это уже записано. Не плодить дубли.
2. **dual-write.** Важное пишу в ДВА места сразу: локальный canonical `.md` + MCP `second_brain-memory.*`. Идемпотентно по sha256 (повтор того же тела — no-op). Локальный `.md` — главный, second_brain — общий слой для команды.
3. **Пиши СРАЗУ, не «потом».** Компакция/конец сессии НЕ выгружают знания автоматически — несохранённое теряется. Зафиксировал — записал.
4. **Пиши в свой scope** (`can_write_scopes`); scope = первая папка пути в vault.

## Что записывать → каким инструментом

| Что (категория) | Инструмент second_brain | Локальный dual-write | Когда |
|---|---|---|---|
| Архитектурные / продуктовые решения | `create_decision_note` | `decisions.md` | принял нетривиальное решение |
| API-контракты, интерфейсы | `create_decision_note` | `decisions.md` | зафиксировал контракт между частями |
| Глобальные правила / выводы «впредь так» | `create_decision_note` (category=rule) | `rules.md`* | вывел правило на будущее |
| Баги + фиксы | `create_error_pattern_note` | — | поймал и устранил баг: что/почему/как не повторить |
| Исходные данные, исследования, опросы, статьи, полезная инфа по проекту | `create_external_note` (+ `source_url`) | `raw/`** | сохранил внешний источник/находку (1–3 абз. выжимка + ссылка) |
| Рабочие заметки, дневной прогресс | `append_daily_log` | `hot/recent.md` | ход работы, мелкие наблюдения |
| **Результаты сессии / перед компакцией** | `create_handoff` | `hot/handoff.md` | конец крупной сессии ИЛИ до сжатия контекста — выгрузить несохранённые решения/ошибки |
| Устаревшее решение | `supersede_decision` | `decisions.md` | прежнее решение больше не верно |
| Черновик/состояние между шагами | `slot_*` | — | эфемерный scratchpad (это НЕ знание) |

\* `rules.md` — RED, прямую правку согласовать с оператором (штатно — ночной learnings-цикл).
\*\* `raw/` — в основном у researcher (long-term store источников).

## Ось субъекта: про человека или про проект

Отдельная ось, **ортогональная** типам выше (decision/error — это «структура»; personal/project — «про кого»). Когда знание относится к субъекту целиком, а не к конкретному решению/процессу:

| Что | Инструмент | Scope | Примеры |
|---|---|---|---|
| Про **человека** (Оператора) | `create_personal_note` | `personal` | ФИО, возраст, навыки, опыт, жизненные ситуации, цели |
| Про **проект / бизнес** | `create_project_note` | `projects` | исходные данные, бухгалтерия, история и структура компании, рабочие чаты и переписка, документация, регламенты, приказы, договора, акты, счета, коммерческие предложения, коммерческая тайна |

Оба scope'а — **общий доступ всей сварме** (read+write). Если у знания есть и субъект, и структура (напр. «карьерное решение Оператора») — выбирай по тому, что важнее искать позже; дубль не нужен (sha256).

## Наивысший приоритет
**Если пользователь прямо просит «запиши / сохрани / запомни» — записать ВСЕГДА**, тип по содержанию, не пропускать и не «решать сам, надо ли».

## Что НЕ записывать
Секреты / токены / пароли · WIP-код · эфемерное состояние сессии · сырые дампы без выжимки.

## Иерархия истины
live-проверка (exec/grep) → second_brain (общий мозг) → git-история → локальная память. Память противоречит проверке — побеждает проверка, память на правку.
