# Создание нового агента (create-agent)

Ты — первый агент (**Developer / Разработчик**). Одна из твоих функций — помогать оператору
заводить НОВЫХ агентов: с собственной ролью, своим Telegram-ботом, той же архитектурой
(второй мозг + канал + автостарт), но другими функциями.

Этот скилл доводит до **рабочего агента с чатом и голосом**. Не вываливай команды —
**проведи оператора за руку**, шаг за шагом, по-человечески.

## Когда применять
Оператор говорит: «заведи нового агента», «добавь агента-X», «нужен ещё один бот-помощник
для …». Если он просто спрашивает про архитектуру — это не сюда.

## Что тебе понадобится от оператора (спрашивай по одному, не списком)
1. **Роль и имя.** Кем будет агент (Коммерсант? Аналитик? SMM? Поддержка?) и как его звать
   (имя → `agent-id` в нижнем регистре). Помоги придумать роль, если оператор не уверен.
2. **Описание роли** одной-двумя фразами — это ляжет в его `CLAUDE.md` (его «душа»).
2a. **Модель Anthropic.** Спроси, на какой модели работать (`opus` / `sonnet` / `haiku`). Подскажи:
   кодеру/координатору — **`opus` (Opus 4.8)**, исследователю — `sonnet`, контент/высокий объём — `haiku`.
   Значение пишется в `settings.json` (поле `model`), а не только в доки. Дефолт для Developer — `opus`.
   Напомни: модель подключается через подписку — Claude Code должен быть авторизован разово командой
   `claude setup-token` (Max/Pro, первая сторона). Без этого агент стартует, но не достучится до модели.
3. **Telegram-бот.** Попроси создать бота: открыть [@BotFather](https://t.me/BotFather) →
   `/newbot` → имя и username → скопировать **токен** вида `1234:AA...`. Объясни, что токен —
   секрет, мы положим его в `channel.env` с `chmod 600`, в git он не попадёт.
4. **Его Telegram user_id** (для allowlist) — узнать у [@userinfobot](https://t.me/userinfobot).
5. **Голос** — нужен ли (по умолчанию да, если в системе есть `GROQ_API_KEY`).

## Ключевые документы — объясни и помоги заполнить
Скаффолдер создаёт воркспейс `~/.claude-lab/<agent-id>/.claude/`. Главные файлы, по которым
стоит пройтись с оператором ПОСЛЕ создания:
- **`CLAUDE.md`** — идентичность агента (роль, характер, язык, как обращаться). Самый важный.
- **`core/rules.md`** — правила поведения и красные зоны (что нельзя без подтверждения).
- **`core/USER.md`** — профиль оператора (чтобы агент знал, на кого работает).
- **`core/AGENTS.md`** — модели и состав команды.
- `@SECONDBRAIN_WRITE_RULES.md` уже подключён — политика записи в общий мозг.

## Как выполнить (механика)
Запусти end-to-end сценарий — он сделает всю рутину и в конце прогонит smoke-тест:

```bash
bash skills/create-agent/new-agent.sh
```

Скрипт по шагам: соберёт параметры → выдаст агенту токен во втором мозге → скаффолдит
воркспейс (`agent-template`, неинтерактивно) → заведёт Telegram-канал из `labops-tg-plugin`
(`channel.env`, привязка плагина) → подключит голос (`groq-voice`) → поставит автостарт
(`systemd` + `watchdog`) → smoke: `recall tools/list`, Telegram `getMe`, активность сервиса.

Можно заранее экспортировать ответы, чтобы скрипт не переспрашивал, напр.:
```bash
AGENT_NAME=Commerce AGENT_ROLE="Коммерсант" LANGUAGE=Russian \
TELEGRAM_BOT_TOKEN=... TELEGRAM_ALLOWED_USER_IDS=... \
bash skills/create-agent/new-agent.sh
```

## После создания
1. Открой с оператором `CLAUDE.md` и `rules.md` нового агента — отредактируйте под роль
   (характер, тон, что агент делает и чего не делает). Это важнее всего остального.
2. Убедись, что smoke-тест зелёный. Если Telegram `getMe` или мозг не прошли — почини
   (токен/хост/`AGENT_BEARER`), не оставляй «полурабочего» агента.
3. Скажи оператору написать новому боту — агент должен ответить (и понять голосовое).

## Принципы
- Секреты (токен бота, bearer) — только в `channel.env`/`agent.env` (chmod 600), НЕ в git, НЕ в чат.
- Один агент — один Telegram-бот, свой `TELEGRAM_WEBHOOK_PORT` (8089, 8090, …), свой state-каталог.
- Если зависимость недоступна (нет второго мозга / tg-plugin) — честно скажи и предложи
  доустановить соответствующий репозиторий, не делай вид, что всё готово.
